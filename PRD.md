## 1. 产品概述

### 1.1 产品定位

**产品名称：** AnimeAvatar（待定）

**一句话描述：** 专注于社媒头像场景的 AI 照片转动漫头像工具，用户上传照片即可生成个性化 anime 风格头像。

**核心差异化：**
- 竞品多为月费订阅（$15-120），本产品**按次收费**，用户门槛更低
- 主打**头像场景**，而非通用图片生成
- 轻量化：**无需注册即可使用**

### 1.2 目标用户

| 用户画像 | 特征 | 需求 |
|----------|------|------|
| Z 世代 / Gen Alpha | TikTok、Instagram 重度用户 | 想要独特的动漫风格头像 |
| 动漫爱好者 | 熟悉 anime 文化 | 尝试把自己变成动漫角色 |
| 社交媒体用户 | 有头像更新需求 | 快速生成好看的头像 |

**目标市场：** 东南亚、北美（英文界面）

### 1.3 成功指标（V1）

- 单次生成成本 < $0.01
- 页面加载 < 3 秒
- 生成完成率 > 60%
- 用户次留 > 20%

## 2. 功能需求

### 2.1 核心功能（MVP 必须）

#### F1: 照片上传
- 支持格式：JPG、PNG、WebP
- 最大文件大小：10MB
- 前端压缩后上传（防止流量浪费）
- 不在服务端存储，图片留存在内存

#### F2: 风格选择
- 提供 **5 种预设风格**：
  1. **Anime Standard** — 标准日漫风
  2. **Manga** — 黑白漫画风
  3. **Ghibli** — 吉卜力动画风
  4. **Cyberpunk** — 赛博朋克风
  5. **Soft Cel-shading** — 柔和赛璐璐风
- 风格以缩略图展示效果预览
- 默认选中 Anime Standard

#### F3: AI 生成
- 调用 Replicate `cjwbw/anything-v4.0` 模型
- img2img 模式，参考上传照片 + anime 风格 prompt
- 生成时间：5-15 秒（预估）
- 前端显示生成进度（polling / websocket）

#### F4: 预览与下载
- 生成完成后全屏预览
- 支持下载原图（1024x1024）
- 提示用户：可用于社媒头像

#### F5: 支付（按次计费）
- **定价：**
  - 单次生成：$0.20（人民币约 ¥1.5）
  - 5 次包：$0.90（约 $0.18/次）
  - 10 次包：$1.70（约 $0.17/次）
- 支付方式：Stripe（信用卡）
- 不做订阅，纯粹的 credits 机制

#### F6: 游客模式
- 无需注册即可使用
- 生成前要求输入邮箱购买 credits
- Credits 记录在本地 cookie（短期），或发邮件链接激活（推荐）

### 2.2 辅助功能（V1 可选）

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 多次生成（1次付费，多张选择） | P2 | 同张照片生成多个结果，用户选最好的 |
| 社交分享 | P3 | 一键分享到 Twitter/X |
| 高清修复（2x放大） | P3 | 额外收费 $0.10 |

### 2.3 不做（V1 明确排除）

- ❌ 用户注册系统
- ❌ 订单历史
- ❌ 退款系统（先做最小化）
- ❌ 客服聊天
- ❌ App（只有 Web）

## 3. 技术方案

### 3.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端框架 | **Next.js 15** (App Router) | 你熟悉的栈 |
| 部署平台 | **Cloudflare Pages** | 免费 CDN，全球加速 |
| AI 服务 | **Replicate API** | img2img 生成 |
| AI 模型 | `cjwbw/anything-v4.0` | $0.0012/张，极低成本 |
| 支付 | **Stripe** | 按次收费，Credits 机制 |
| 存储 | **无** | 不存图片，纯内存处理 |
| 域名 | 待定 | 建议 .com 英文名 |

### 3.2 前端架构

```
src/
├── app/
│   ├── page.tsx              # 首页（上传+风格选择+生成）
│   ├── result/page.tsx       # 结果预览页
│   ├── pricing/page.tsx      # 定价页
│   └── api/
│       └── generate/route.ts # 调 Replicate 的接口
├── components/
│   ├── UploadZone.tsx        # 上传区
│   ├── StyleSelector.tsx     # 风格选择器
│   ├── GenerationStatus.tsx  # 生成进度
│   └── ResultPreview.tsx     # 结果预览
└── lib/
    ├── replicate.ts           # Replicate SDK 封装
    └── stripe.ts              # Stripe 逻辑
```

### 3.3 API 设计

**POST /api/generate**

Request:
```json
{
  "image": "data:image/jpeg;base64,...",  // 压缩后的 base64
  "style": "anime_standard"               // 风格 key
}
```

Response (success):
```json
{
  "success": true,
  "image": "data:image/png;base64,..."    // 生成的图片
}
```

Response (error):
```json
{
  "success": false,
  "error": "INSUFFICIENT_CREDITS" | "GENERATION_FAILED"
}
```

### 3.4 支付流程

```
用户点击"生成" 
  → 检查 cookie credits
  → 不足则弹窗购买
  → Stripe Checkout
  → 支付成功 → 增加 credits 到 cookie
  → 继续生成
```

Credits 存储方案（简化版）：
- 方案 A：Cookie 存储（简单，但可作弊）
- 方案 B：Stripe Customer + Credits 记录（更完整）

**建议 V1 用方案 A 快速上线，后续再加 B。**

## 4. UI/UX 设计方向

### 4.1 页面结构

```
首页（单页）
├── Hero 区域：标题 + 一句话描述
├── 上传区（拖拽 / 点击上传）
├── 风格选择器（横向滑动，5个风格卡片）
├── 生成按钮
└── Footer：定价入口 + 版权
```

### 4.2 设计风格

- **风格：** 简洁现代 + anime 元素点缀
- **主色调：** 浅色背景 + 紫色/粉色渐变按钮（anime 常见配色）
- **字体：** 无衬线字体，干净易读
- **参考网站：** ImagineArt、PhotoAI.com

### 4.3 响应式

- 移动端优先（大多数用户从手机访问）
- 桌面端：最大宽度 800px 居中
